

## Add Pull Request Template

Create `.github/PULL_REQUEST_TEMPLATE.md` with standard sections that auto-populate when contributors open a PR.

### Content Structure

1. **Description** -- free-text area for what changed and why
2. **Type of Change** -- checkbox list (bug fix, new feature, breaking change, docs update, refactor)
3. **Checklist** -- pre-merge checks (builds pass, tests added/updated, follows existing patterns, no hardcoded colors, TypeScript fully typed)
4. **Related Issues** -- link to relevant GitHub issues

Single file creation, no other changes needed.
