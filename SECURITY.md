# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Ping, please report it responsibly. **Do not open a public issue.**

Email **security@socialocca.io** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

## Response Timeline

| Step | Timeframe |
|------|-----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Varies by severity |

## Scope

**In scope:**
- The Ping web application (`pingsocialocca.lovable.app`)
- Backend APIs and edge functions
- Authentication and authorization flaws
- Data exposure or injection vulnerabilities

**Out of scope:**
- Third-party services and dependencies (report upstream)
- Social engineering or phishing attacks
- Denial-of-service attacks
- Issues in forked or modified copies

## Disclosure

We ask that you give us reasonable time to address the issue before any public disclosure. We're happy to credit researchers who report valid vulnerabilities.

Thank you for helping keep Ping secure.
